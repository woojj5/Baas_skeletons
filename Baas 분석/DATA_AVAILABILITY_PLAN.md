# 상세 분석 모달 데이터 가용성 계획

## 데이터 가용성 판단 기준

### 주행 관련 데이터
- **조건**: `drive_count > 0` 또는 `parking_count > 0`
- **데이터 없음**: `drive_count === 0 && parking_count === 0`
- **영향받는 항목**: 효율, 온도, 셀 밸런스, 주행

### 충전 관련 데이터
- **조건**: `fast_charge > 0` 또는 `slow_charge > 0`
- **데이터 없음**: `fast_charge === 0 && slow_charge === 0`
- **영향받는 항목**: 충전

## 각 상세 분석 모달별 계획

### 1. 효율 상세 분석 (`openEfficiencyDetail`)
- **필요 데이터**: 주행 구간 데이터 (주행거리, 전력량)
- **데이터 가용성 확인**:
  - `drive_count > 0` 또는 `parking_count > 0` → 정상 표시
  - `drive_count === 0 && parking_count === 0` → "주행 관련 데이터가 없습니다" 메시지 표시
- **표시 내용**:
  - 데이터 있음: 효율 시계열, 주행거리/전력량 차트, 변환 그래프, 점수 계산 결과
  - 데이터 없음: 안내 메시지 및 기본 정보만 표시

### 2. 온도 상세 분석 (`openTemperatureDetail`)
- **필요 데이터**: 주행 구간 데이터 (온도 데이터)
- **데이터 가용성 확인**:

  - 데이터 없음: 안내 메시지 및 기본 정보만 표시

### 3. 셀 밸런스 상세 분석 (`openCellBalanceDetail`)

## 구현 방법

### JavaScript 함수 수정
각 `openXXXDetail()` 함수에서:
1. `currentVehicleDetailData.section_counts` 확인
2. 필요한 구간 데이터가 있는지 확인
3. 데이터 없으면:
   - 모달은 열되, 안내 메시지 표시
   - 차트는 그리지 않음
   - "데이터 없음" 상태 표시
4. 데이터 있으면:
   - 정상적으로 모든 차트와 정보 표시

### 안내 메시지 템플릿
- 주행 관련: "주행 구간 및 주차 구간 데이터가 없어 주행 관련 상세 분석을 제공할 수 없습니다."
- 충전 관련: "급속 충전 및 완속 충전 데이터가 없어 충전 관련 상세 분석을 제공할 수 없습니다."
